Below is a detailed UI plan for Nour, focused on making the interface feel polished, Arabic-first, operational for merchants, and trustworthy for shoppers.

UI Direction
Nour should feel soft, modern, and fashion/cosmetics-aware without becoming decorative or slow. Storefronts can be expressive and visual. Merchant admin should be quieter, denser, and easier to scan.

The visual system should balance:

Warm beauty/fashion identity
Arabic-first typography and RTL spacing
Clear COD/shipping/payment trust signals
Efficient merchant operations
Strong mobile usability
Accessible contrast and touch targets
1. Design System Foundation
Create a stricter UI foundation before redesigning individual screens.

Core tokens:

Background: warm off-white, but not overly beige
Surface: white/card
Text primary: deep neutral
Text secondary: muted neutral
Primary: deep rose/magenta
Accent: restrained gold
Success: green
Warning: amber
Danger: red
Info: blue
Border: soft neutral
Focus ring: primary with visible contrast
Typography:

Arabic UI: Cairo or Tajawal
English UI: Inter
Use one consistent UI font family per language.
Avoid serif-heavy admin UI.
Reserve editorial serif styling for storefront heroes and product merchandising only.
Admin headings should be compact, not hero-sized.
Recommended scale:

Page title: 28-32px
Section title: 18-22px
Card title: 15-17px
Body: 14-16px
Meta/helper text: 12-13px
Table/list text: 13-14px
Spacing:

Base unit: 4px
Main gaps: 8, 12, 16, 24, 32
Admin screens should use tighter spacing.
Storefront sections can use larger spacing.
Radius:

Admin cards/buttons: 8-12px
Storefront imagery/cards: 12-20px when visually useful
Avoid excessive rounded-3xl in admin workflows.
Use pill buttons only for shopper CTAs or compact filters, not everywhere.

12. Empty, Loading, Error States
Create a consistent state system.

Loading:

Skeletons shaped like final layout.
Avoid generic spinners for whole pages unless necessary.
Empty states:

State what is missing.
Explain why it matters.
Provide one primary action.
Examples:

No products:
“لم تضيفي منتجات بعد”
CTA: “إضافة أول منتج”
No orders:
“لا توجد طلبات بعد”
CTA: “معاينة المتجر”
No shipping zones:
“حددي مناطق وأسعار الشحن”
CTA: “إضافة منطقة شحن”
Error states:

Plain Arabic message.
Recovery action.
Avoid raw server errors.
Keep field-level errors close to fields.
13. Accessibility And QA
UI acceptance checklist:

All controls keyboard reachable.
Focus states visible.
Buttons have accessible names.
Icon-only buttons have aria-label and tooltip.
Touch targets at least 44px on mobile.
Normal text contrast at least 4.5:1.
Large text contrast at least 3:1.
Text scales to 200%.
No text overlap on mobile.
RTL and LTR both checked.
Phone/email/URL fields use LTR.
No action depends only on hover.
Motion respects reduced-motion preference.

Yes, we can use multiple agents effectively, but I’d use them like a small product squad, not a crowd. The main risk with UX/UI work is fragmented taste and conflicting edits, so I’d keep one “lead” thread integrating decisions and give agents tightly scoped lanes.

Best Operating Model
Use 4-6 agents max:

UX Lead / Integrator

Owns the full journey map.
Keeps UX and UI plans aligned.
Resolves tradeoffs.
Reviews final implementation quality.
Shopper Flow Agent

Owns storefront, product detail, cart, checkout, confirmation.
Focus: COD trust, Arabic copy, mobile conversion.
