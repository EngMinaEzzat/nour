Sprint Plan To Fully Finish

Sprint 6A: UI Foundation

Create shared components: StatusBadge, StateBlock, AdminPageHeader, AdminToolbar, AdminTable, IconButton.

Add admin density classes: compact headings, 8-12px radius defaults, table spacing.
Acceptance: new or changed admin screens use shared components, no new one-off badge/status styles.
Sprint 6B: App Shell + Global States

Add nav badges from real data: pending orders, low stock, returns/follow-up, failed contact if available.
Add accessible tooltips/labels to shell actions.
Build shared loading/empty/error states and migrate cart, orders, products, dashboard, builder.
Acceptance: shell guides merchant work, no major icon-only button lacks accessible name.
Sprint 6C: Shopper UI Completion

Finish product detail accessibility: swatches, variant disabled states, quantity controls, review display rules.
Strengthen checkout trust strip and Paymob visibility gating.
Polish cart copy and totals consistency.
Acceptance: shopper can browse, choose variant, cart, checkout with keyboard/mobile and no fake social proof.
Sprint 6D: Merchant Command Center

Redesign dashboard as operational dashboard, not decorative cards.
Finish orders with city/area/date filters, SLA/age severity, failed-contact queue, courier/tracking columns.
Improve order detail hierarchy: top action bar, fulfillment vs communication separation.
Acceptance: merchant can know “what needs action now” within 10 seconds.
Sprint 6E: Product Admin Completion

Complete filters: active, hidden, out of stock, low stock, has variants, missing image, no category.
Add quick stock edit and bulk actions where API supports it.
Convert product form into tabs/accordion: Details, Images, Price, Variants, Inventory, SEO/Visibility.
Acceptance: visual review and fast operations both work without relying on hover.
Sprint 6F: Builder + Platform Admin

Builder: contrast preview for text/buttons/cards, preset warnings, publish readiness modal.
Platform: operator tables, health/risk filters, audit/action log surface, incident/error summary.
Acceptance: builder prevents unreadable themes; platform admin feels like an operations console.
Sprint 6G: Full QA Pass

Run the full visual QA matrix from the original UI plan on mobile and desktop.

Fix all critical/serious findings before calling the UI plan complete.
Acceptance: documented pass/fail matrix with screenshots or notes for every planned screen.
