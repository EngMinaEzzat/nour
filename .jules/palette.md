## 2025-05-15 - [Accessible Icon-Only Buttons and Localized Feedback]
**Learning:** In an RTL (Right-to-Left) and localized (Arabic) storefront, icon-only buttons (like Wishlist hearts or Star ratings) are frequently used for visual cleanliness, but they are completely invisible to screen readers without explicit `aria-label` attributes. Furthermore, silent state changes (like adding to a bag) need prominent, localized feedback (Success Toasts) to confirm the action, especially when the cart drawer is not automatically opened.
**Action:** Always provide `aria-label` for icon-only buttons and use success toasts for critical async actions like "Add to Bag" to ensure accessibility and clear user feedback.
## 2024-06-27 - Icon-only buttons Accessibility in Custom Storefronts
**Learning:** Custom storefront themes like GlowGridStorefront often omit ARIA labels for icon-only buttons (like Cart or Add to Cart), which are separated from the main UI component system. Additionally, ligature text used inside decorative icons (e.g., `<span class="material-symbols-outlined">shopping_bag</span>`) is read by screen readers if it is not explicitly hidden with `aria-hidden="true"`.
**Action:** When auditing or implementing custom storefront templates, explicitly add `aria-label={t(...)}` to icon-only buttons and hide the visual ligature text within using `aria-hidden="true"`. Ensure `react-i18next` (`useTranslation`) is imported to support internationalized ARIA labels.
