## 2024-06-04 - Accessible Accordion Details Buttons\n**Learning:** When using standard `<button>` tags as simple toggles for details/accordions (e.g., 'Show Details'), without a headless UI library, they often lack proper `aria-expanded` and `aria-controls` attributes, leaving screen reader users without context about what the button controls or its current state.\n**Action:** Always ensure accordion-style expand/collapse buttons have an `aria-expanded` state tied to the toggle boolean, an `aria-controls` pointing to a unique `id` on the expandable content container, and clear keyboard focus styles.
## 2026-05-23 - Accessible Icon Buttons in Components\n**Learning:** When using components like `Lucide` icons directly as button content, missing `aria-label` and `title` attributes make them inaccessible to screen readers and difficult to understand without tooltips. Translations for accessibility strings should be stored correctly in JSON locale files.\n**Action:** Always provide localized `aria-label` and `title` tags for icon-only interactive elements and ensure all new strings are translated across supported locales (e.g., Arabic and English).
