Title: 🧪 [Testing Improvement] Add CSRF Configuration Tests

Description:
🎯 **What:** The testing gap addressed
This PR addresses the missing tests for the `doubleCsrfProtection` configuration in `artifacts/api-server/src/lib/csrf.ts`.

📊 **Coverage:** What scenarios are now tested
- Validates CSRF cookie configuration attributes (e.g., `secure: false`, `sameSite: "lax"`) correctly fallback in development (`NODE_ENV !== "production"`).
- Asserts that `secure: true` and `sameSite: "none"` are applied when in production.
- Ensures the default `SESSION_SECRET` fallback acts as expected when `process.env.SESSION_SECRET` is omitted.
- Validates request value extraction for the session ID and CSRF token (`getSessionIdentifier` and `getCsrfTokenFromRequest`).
- Confirms the array `CSRF_EXEMPT_PATHS` is accurately respected by the `isCsrfExempt` helper.

✨ **Result:** The improvement in test coverage
The application's `lib/csrf.ts` module is completely covered, improving confidence around crucial security setup details and reducing the likelihood of regression in webhook configurations or HTTP cookie directives.
