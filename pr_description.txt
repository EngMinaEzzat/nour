🎯 What
Replaced raw `console.log` and `console.error` calls across operational scripts with a dedicated logger utilizing formatted text and standard terminal color codes.

💡 Why
This improves the maintainability and readability of script execution output by providing standard log levels (`info`, `success`, `error`, `warn`, `skip`) and a consistent structured logging interface across operations.

✅ Verification
- Ensured script compatibility by testing imports with `tsx`.
- Ran scripts (like `smoke.mjs` and `phase7-load-smoke.mjs`) locally to confirm expected colored output parsing and execution.
- Passed workspace `pnpm run typecheck` and workspace builds successfully.

✨ Result
Logs are now clearly identifiable visually and structured systematically without adding complex dependencies to standalone scripts.
