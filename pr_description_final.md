### 💡 What:
Replaced the fixed grid-based sizes input interface with dynamic arbitrary size inputs for product variants, accurately modeling the user's requested layout where sizes and corresponding opening quantities can be created on-the-fly.
### 🎯 Why:
To allow merchants to list variants with arbitrary sizes or measurements rather than locking them into preset clothing sizes like XS, S, M, L.
### 📸 Before/After:
Before: Fixed SIZES grid requiring clicks to toggle.
After: Flex rows supporting custom text size values plus their initial stock quantities.
### ♿ Accessibility:
Properly retained and applied translation variables and `aria-label` equivalent hints.
